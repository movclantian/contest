CCKS 2026 OneEval 赛制介绍
一、赛题与任务背景
本次评测由东南大学组织，依托阿里巴巴天池平台开展，旨在构建兼具挑战性、可解释性与可控难度的新一代评测标准。任务核心理念为以失败驱动评测，通过挖掘模型在处理复杂知识时的错误案例，精准探测大语言模型在知识密集型推理任务中的能力边界。

评测涵盖三种模态的复杂推理场景：

Text（文本）推理：基于文本信息在跨文档、跨维度数据间整合证据并推断逻辑路径
KG（知识图谱）推理：考察模型在多步逻辑推导、知识源不完整及干扰信息环境下的抗干扰能力
Table（表格）推理：模型需从表格结构中识别显性信息并整合推断隐含的推理结论
评测通过以下两个维度对模型施加极限压力：

知识压力 (K-Stress)：在刻意缺失关键事实的前提下，检验模型在知识不完备情境下的补全与稳健性
推理压力 (R-Stress)：引入虚构实体、多维干扰项及复杂关系链，考察模型在长链推理中的逻辑严谨性
二、参赛与组队规则
开放对象：大赛面向全社会开放，选手通过阿里云天池平台完成个人信息注册即可报名
队伍规模：支持单人参赛或组队，每支队伍不超过3人
组队限制：每位选手只能加入一支队伍，选手报名及组队变更截止时间为7月3日
实名认证：全体成员须在7月3日 23:59:59前完成支付宝实名认证，未认证团队将无法进行后续比赛
三、系统评测与指标
评测流程：选手在本地调试算法后在线提交结果，系统在评测期间每天提供多次评测机会并实时返回成绩
排名规则：排行榜按照评测指标从高到低排序，系统将选择参赛队伍在本赛段的历史最优成绩进行展示
评测指标：采用三指标融合评分策略，由精确匹配得分 (Sem)、F1 得分 (Sf1) 与 ROUGE-L 得分 (Srouge) 取均值构成单条实例得分

本评测任务的核心理念是“以失败驱动评测”：通过深度挖掘模型在处理复杂知识时的错误案例，构建出更具针对性、更具挑战性且难度可控的新测试实例。本次评测聚焦知识密集型推理的复杂性：模型不仅需要从文本、图谱或表格中识别显性信息，还必须在跨文档、跨关系链和跨维度数据之间整合证据，并在此基础上推断隐含的推理路径。参评模型需应对基于文本（Text）、知识图谱（KG）和表格（Table）等多种模态的复杂推理场景，特别是在知识源不完整或存在干扰信息的真实检索增强生成（RAG）环境中，考验模型在知识补全、多步逻辑推导以及对细微陷阱的抗干扰能力。

答题须知
一、赛题数据说明
本赛题提供统一的赛题数据文件 contest_data.json，包含 250 道题目，涵盖三种任务类型：

任务类型	数量	说明
knowledge_graph	100 题	知识图谱问答：根据给定的知识图谱三元组推理回答问题
multi_hop_qa	100 题	多跳文本问答：根据给定的多段文本上下文回答问题
table_qa	50 题	表格问答：根据给定的结构化表格数据回答问题
赛题数据格式
每条赛题为一个 JSON 对象，包含以下字段：

字段	类型	说明
id	int	题目唯一编号（1–250），答题时需在答案中对应此编号
task_type	string	任务类型：knowledge_graph / multi_hop_qa / table_qa
question	string	问题文本
input	string	知识图谱三元组（仅 knowledge_graph 类型存在）
contexts	array	文本段落数组（仅 multi_hop_qa 类型存在），每个元素包含 title、sentences、paragraph 等字段
table	object	表格数据（仅 table_qa 类型存在），包含 header（表头）和 rows（数据行）
注意：input、contexts、table 三个字段根据 task_type 只出现一个，互斥。

二、答题要求
选手需针对每道题目，根据提供的上下文信息（知识图谱三元组 / 文本段落 / 表格数据）进行阅读理解与推理，给出最终答案。

答案格式
所有答案统一以 字符串 形式提交
答案应简洁明了，直接回答问题即可
不需要附加推理过程或解释
三、提交格式（重要！）
文件格式
提交文件名为 submit.jsonl，采用 JSON Lines 格式，每行一个 JSON 对象：

{"id": 1, "answer": "你的答案"}
{"id": 2, "answer": "你的答案"}
{"id": 3, "answer": "你的答案"}
...
字段说明
字段	类型	必填	说明
id	int	✅	题目编号，必须与赛题数据中的 id 一一对应
answer	string	✅	你的答案，字符串类型
提交示例
{"id": 1, "answer": "Setup"}
{"id": 2, "answer": "Nick Cannon"}
{"id": 100, "answer": "William Scott Vare"}
{"id": 101, "answer": "Capote"}
{"id": 200, "answer": "Odran Vell"}
{"id": 201, "answer": "6"}
{"id": 250, "answer": "Vantrel Foxes"}
提示：题目顺序可以任意打乱，评测系统按 id 进行匹配，不依赖行序。
